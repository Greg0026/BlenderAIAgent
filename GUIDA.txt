1. REQUISITI
═══════════════════════════════════════════════════════════════

  - Python 3.9+
  - Blender 3.0+ (impostare path in .env)
  - Connessione internet (API LLM)
  - Almeno 4 GB RAM libera
  - GPU consigliata per Vision Review (opzionale)


2. INSTALLAZIONE
═══════════════════════════════════════════════════════════════

  2.1 Installa dipendenze:
      pip install -r requirements.txt

  2.2 Copia e configura il file .env:
      cp .env.example .env

      Variabili OBBLIGATORIE (almeno una API key):
        NVIDIA_API_KEY         Chiave API NVIDIA NIM
        OPENROUTER_API_KEY     Chiave API OpenRouter
        OPENCODE_ZEN_API_KEY   Chiave API OpenCode Zen (alternativa a NVIDIA/OpenRouter)

      Variabile OBBLIGATORIA:
        BLENDER_PATH           Percorso assoluto eseguibile Blender

      Se usi OPENCODE_ZEN_API_KEY, imposta anche:
        LLM_BASE_URL           URL API (es. https://api.opencode.ai/v1)

      Variabili OPZIONALI principali:
        LLM_MODEL_ID           Modello primario (default: z-ai/glm-5.1)
        LLM_VISION_MODEL       Modello per revisione visiva
        VISION_REVIEW_ENABLED  true/false
        BLENDER_TIMEOUT        Secondi massimi esecuzione Blender (default: 120)

  2.3 Verifica Blender:
      blender --version
      Se non in PATH, impostare BLENDER_PATH in .env


3. USO BASE
═══════════════════════════════════════════════════════════════

  3.1 Singolo prompt:
      python main.py
      (usa il prompt di default: portapenne 3 scomparti)

      Con prompt personalizzato:
      python main.py -p "crea un vaso floreale con base esagonale, altezza 15cm"
      python main.py --prompt "crea un portapenne" -o output.py

      Output: script Blender salvato in 3dtest.py (o -o).
      Eseguire con:
      blender --background --python 3dtest.py

  3.2 Batch mode (generazione automatica continua):
      python afk.py

      Variabili d'ambiente per batch:
        BATCH_MAX_JOBS=10        Numero massimo job (vuoto = illimitato)
        BATCH_PAUSE=15           Secondi tra un job e l'altro
        BATCH_OUTPUT_DIR=output  Directory di output

  3.3 Costruire il corpus VectorDB (per migliorare la RAG):
      python corpus_builder.py -v
      python corpus_builder.py --index-only   (solo indicizzazione)
      python corpus_builder.py --load-only    (riusa corpus.jsonl esistente)


4. ARCHITETTURA DELLA PIPELINE
═══════════════════════════════════════════════════════════════

  Prompt → [F1] Enhancement → [F1.5] Math Planner → [F2] Code Generation
      ┌────────── VISION LOOP (max 8x) ──────────┐
      │  [F3A] Morph Review → [F3B] Printability   │
      │    ┌──────── FIX LOOP (max 6x) ───────┐    │
      │    │  [F3.5] Static Analysis → [F6] Fix │    │
      │    │  [F4]   Blender Execute  → [F6] Fix │    │
      │    │  [F4]   Mesh Validation   → [F6] Fix │    │
      │    └─────────────────────────────────────┘    │
      │  [F4.5] Vision Review (LLM vision) → [F6-VIS] │
      └───────────────────────────────────────────────┘
      → Script Python Blender finale → opzionale STL


5. DESCRIZIONE FASI
═══════════════════════════════════════════════════════════════

  Fase        Input     Output        Descrizione
  ─────────────────────────────────────────────────────────
  F1          Prompt    Specifica     Traduce richiesta in
                          tecnica       parametri numerici + archetipo
  F1.5        Spec      Piano         Formule r(z), punti di
                          matematico    controllo, primitive
  F2          Spec+     Script bpy    Genera primo script Blender
              piano+docs               (con RAG VectorDB)
  F3A         Script    Script        Verifica fedeltà morfologica
              +spec       corretto      al prompt
  F3B         Script    Script        Verifica stampabilità 3D
              +spec       corretto      (manifold, spessore, API)
  F3.5        Script    Errori/       AST linter: sintassi, moduli
                          Warning       pericolosi, euristiche 3D print
  F4          Script    Output        Esecuzione headless Blender +
                          Blender+     validazione geometrica mesh
                          validazione   (non-manifold, volume zero)
  F4.5        Script    Report        Render 4 viste ortografiche +
              +spec       visivo        LLM vision valuta estetica
  F6          Script+   Script        Correzione errore runtime/
              errore      corretto      static con VectorDB retrieval
  F6-VIS      Script+   Script        Correzione problemi
              report      corretto      visivi/estetici


6. ARCHETIPI
═══════════════════════════════════════════════════════════════

  A  Rivoluzione       Bicchieri, vasi, ciotole      r(z) parametrico
  B  Estrusione/Loft   Manici, profili               Curve NURBS + sweep
  C  Boolean Composite  Scatole, carter               Primitive + booleane
  D  Voronoi/Fractal    Paralumi, sculture            bmesh + noise + subdiv
  E  Ibrido             Oggetti complessi             Mix di approcci


7. MECCANISMI DI STABILITA'
═══════════════════════════════════════════════════════════════

  - OscillationDetector: rileva pattern A→B→A→B e forza uscita
  - ErrorHistory:   tiene traccia errori, evita fix già falliti
  - Fallback script: se la pipeline fallisce, genera cubo base
                     con Solidify + Weld + Subsurf
  - Model cascade:  se modello primario fallisce, prova fallback
  - Weak model filter: rimuove modelli <7B parametri
