━━━ GEOMETRIC ARCHETYPES (Complete Guide) ━━━
Choose ONLY ONE for each object. Do not force the object into the wrong archetype:
each archetype has specific requirements and formalisms. Mixing them generates internally
contradictory specifications that the review loop will never be able to resolve.

────────────────────────────────────────────────────
A. SURFACE OF REVOLUTION
────────────────────────────────────────────────────
Objects: vases, bottles, candles, rings, bowls, glasses, columns, spherical
          lampshades, objects with full axial symmetry (360 degrees).
Method: parametric r(z) profile generated by revolution using bmesh.
        Each vertex is at coordinates (r(z)*cos(theta), r(z)*sin(theta), z).
Parameters:
  N_SEGS >= 128  (number of angular segments — horizontal resolution)
  N_RINGS >= 64  (number of vertical loops — profile resolution)
  Profile: r(z) = r_base * (1 + A * sin(pi * z / H)) with explicit A and H
           or interpolated spline control points
  Thickness: Solidify offset=-1.0, thickness >= WALL_MM/1000
  Bottom: fan fill (bmesh.ops.poke) or grid fill on the loop at z=0
Avoid: constant radius across the entire profile (if the plan requires modulation);
       N_SEGS < 64 (produces visible polygons on the curved surface).

────────────────────────────────────────────────────
B. PRISMATIC / MECHANICAL SOLID
────────────────────────────────────────────────────
Objects: boxes, brackets, supports, enclosures, bases, junction boxes,
          frames, objects with predominantly flat faces and sharp edges.
Method: Start from a block/polygonal profile (even a primitive as a base),
        then add bevel/fillet on edges and features via booleans/inset.
Parameters:
  Base block dimensions: L x W x H in mm
  Bevel radius: explicit in mm for each edge
  Bevel segments >= 16 per curved arc
  Features: holes (position, diameter), ribs (position, L x W x H),
            fins (position, thickness, height)
  Walls: Solidify or thickness managed in bmesh construction
  r(z) and >=128 segments are NOT needed: those counts apply only to
    curved fillets, not to the entire solid.
Avoid: primitive left completely unchanged (without features, without
       bevel, without thickness); requiring >=128 segments for flat faces.

────────────────────────────────────────────────────
C. GEAR / INVOLUTE PROFILE
────────────────────────────────────────────────────
Objects: gears, cogwheels, racks, pinions.
Method: explicit involute tooth profile with bmesh, extruded along the axis.
Involute tooth formula:
  x = r_b * (cos(theta) + theta * sin(theta))
  y = r_b * (sin(theta) - theta * cos(theta))
  where r_b = base radius, theta from 0 to theta_max
Parameters: module, number of teeth, pressure angle (default 20 degrees),
           tooth thickness, pitch diameter, tip diameter, root diameter.

────────────────────────────────────────────────────
D. HELIX / THREAD
────────────────────────────────────────────────────
Objects: screws, springs, spirals, spiral staircases, propellers.
Method: bmesh vertices generated using parametric equations:
  x(t) = R(t) * cos(t)
  y(t) = R(t) * sin(t)
  z(t) = pitch * t / (2 * pi)
  for t from 0 to N_turns * 2 * pi
Parameters: radius R, pitch (lead per turn), N_loop (number of turns),
           section thickness, N_SEGS per turn >= 32.

────────────────────────────────────────────────────
E. PART ASSEMBLY
────────────────────────────────────────────────────
Objects: objects composed of multiple distinct parts, each an instance of one
          of archetypes A-D, joined or subtracted with safe boolean operations.
Method: build each part separately with the corresponding archetype,
        then combine with Boolean UNION (union) or DIFFERENCE (subtraction).
Requirements:
  List each sub-component with its own archetype and parameters
  Specify relative positioning coordinates
  Specify boolean operation for each union
  Verify that interfaces between parts are manifold before the boolean
  Unions can also be done with Join (bpy.ops.object.join) if the
    parts do not overlap; subtractions require Boolean Difference.

────────────────────────────────────────────────────
GENERAL RULE
────────────────────────────────────────────────────
A raw primitive (cube, sphere, cylinder WITHOUT modifications) is FORBIDDEN because
it does not constitute a printable object. A primitive used as a BASE and then
MODIFIED with booleans, solidify, subdivision, bevel, or deformation is
ACCEPTABLE and indeed recommended for archetype B.
