━━━ CATALOGO ERRORI E SOLUZIONI SPECIFICHE ━━━

ERR-1. "poll() failed" / "context is incorrect" / "Cannot run in background mode":
   Causa: bpy.ops.* chiamato senza contesto corretto.
   Soluzione: bpy.context.view_layer.objects.active = obj
              bpy.ops.object.mode_set(mode='OBJECT')
   Verifica che l'oggetto esista ancora prima di attivarlo.

ERR-2. "ReferenceError: StructRNA of type Object has been removed":
   Causa: l'oggetto e' stato eliminato ma ancora referenziato.
   Soluzione: riordina il ciclo di vita oggetti.
   Usa "if obj.name in bpy.data.objects:" prima di operare su un oggetto.

ERR-3. "AttributeError: 'NoneType' object has no attribute '...'":
   Causa: un oggetto non esiste quando viene usato.
   Soluzione: aggiungi "if obj is None: raise RuntimeError(...)" dopo ogni creazione.

ERR-4. API obsoleta (Blender 3.x+):
   obj.select = True           ->  obj.select_set(True)
   scene.objects.link(obj)     ->  bpy.context.collection.objects.link(obj)
   context.scene.objects.active -> bpy.context.view_layer.objects.active
   bpy.ops.object.modifier_apply(apply_as='DATA') -> modifier=mod.name

ERR-5. "bmesh: mesh is in editmode":
   Causa: bm.to_mesh() chiamato mentre la mesh e' in EDIT mode.
   Soluzione: bpy.ops.object.mode_set(mode='OBJECT') prima di bm.to_mesh().

ERR-6. "Modifier is disabled, skipping apply":
   Causa: modifier con show_viewport = False.
   Soluzione: mod.show_viewport = True prima di modifier_apply.

ERR-7. "KeyError: modifier not found":
   Causa: mod.name non corrisponde alla stringa usata in modifier_apply.
   Soluzione: verifica che mod.name sia lo stesso usato in
   bpy.ops.object.modifier_apply(modifier=mod.name).

ERR-8. "ZeroDivisionError" / "ValueError" in math:
   Causa: divisione per zero in formula geometrica.
   Soluzione: proteggi ogni denominatore con "if H == 0: H = 0.001".

ERR-9. "Boolean solver error" / geometria non manifold dopo booleana:
   Causa: mesh del cutter non manifold, o scale non applicata, o mesh condivisa.
   Soluzione:
   1. bmesh.ops.remove_doubles PRIMA della booleana
   2. transform_apply su cutter e body PRIMA della booleana
   3. cutter con mesh DEDICATA (usa .copy() se necessario)
   4. solver='EXACT', overshoot >= 20%

ERR-10. "Nothing selected" / "Cannot execute" su operatore:
   Causa: select_set chiamato senza deselezionare prima.
   Soluzione: bpy.ops.object.select_all(action='DESELECT') PRIMA di
   obj.select_set(True).
