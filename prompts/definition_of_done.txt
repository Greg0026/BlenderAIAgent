━━━ DEFINITION OF DONE (Unique Completion Criterion) ━━━
These criteria are SHARED by F3A (Morphological Review), F3B (Printability Review),
F4.5 (Vision Review) and F6-VIS (Vision Fix). If ALL are satisfied, the script is
COMPLIANT and must be returned unchanged — any further rewriting is a
possible source of regressions and infinite non-convergence loops.

────────────────────────────────────────────────────
D1 — CORRECT ARCHETYPE
The silhouette matches the ARCHETYPE declared in the algorithmic plan: no more,
no less. No requirements borrowed from a different archetype (e.g., do not
require a sinusoidal r(z) formula for an object classified as B-prismatic).
Is the archetype the right one for the requested object? If not, the technical
specification (F1) made a mistake — but the fix must be made there, not in the script.

────────────────────────────────────────────────────
D2 — PARAMETERS RESPECTED
The formulas/numerical parameters from the algorithmic plan are implemented with the same
values (+/-5% tolerance only for unit conversion rounding, not for formula
changes). The FROZEN PARAMETERS of the technical specification are all present and
unchanged. No placeholders like "N" or "R" in the code.

────────────────────────────────────────────────────
D3 — MANIFOLD MESH
The mesh is manifold: zero free edges, zero non-manifold faces, zero isolated vertices,
normals consistent pointing outward, bottom closed (no free edge on the z=0 plane).
Watertight: if immersed in water, water does not enter (no holes).

────────────────────────────────────────────────────
D4 — WALL THICKNESS
The wall thickness is >= WALL_MM (1.2 mm for FDM, 0.8 mm for SLA) wherever an
intentional cavity is expected. Verified by Solidify Modifier (thickness >=
WALL_MM/1000, offset=-1.0) or by direct geometric calculation. The use of
uniform scale to generate cavities is not allowed.

────────────────────────────────────────────────────
D5 — ERROR-FREE EXECUTION
The script runs in Blender (headless, --background) without exceptions and produces
the "PrintReady_<name>" object. The resulting mesh is visible in the scene and has
actual vertices (not zero vertices).

────────────────────────────────────────────────────
WHEN TO STOP (FUNDAMENTAL RULE) ━━━
If D1-D5 are satisfied, there is NO further level of "perfection" to
achieve in this pipeline. Looking for other stylistic defects, micro-optimizations,
or "aesthetic improvements" beyond D1-D5 is the most common cause of:
  - Non-convergence loops (F3A finds a new defect every time)
  - Oscillations (A->B->A->B between two versions of the script)
  - Regressions (a visual fix undone by a subsequent rewrite)
When in doubt between "compliant" and "improvable", COMPLIANCE WINS OVER REWRITING.
